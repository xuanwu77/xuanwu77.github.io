什么是CSS hack？

不同的浏览器对某些CSS代码解析会存在一定的差异，因此会导致不同浏览器下给用户展示的页面效果不一样，碰到这样的情况就需要开发人员通过针对不同的浏览器对应写出不同的CSS代码，这种技术有个专门的名称就是CSS Hack。

对于前端开发人员来说，应当尽可能地不使用CSS的hack技术来实现需求，说是不太友好，那咱么就听他的，少用。不过某些情况下，为了顾及用户体验实现向下兼容，不得已需要使用hack。所以这篇文章呢，就来总结下CSS hack的一些用法。

CSS hack的分类

CSS Hack有三种实现方式，它们分别是：

  • HTML头部的hack(条件注释法)
  • CSS类内部的hack(类内属性前缀法)
  • CSS选择器的hack(选择器前缀法)

这种方法是在CSS样式属性名前加上一些只有特定浏览器才能识别的前缀，或是在属性值后加上特定后缀，来区分各浏览器的页面展示效果。
