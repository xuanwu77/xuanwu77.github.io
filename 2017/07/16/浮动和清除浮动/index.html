浮动和清除浮动

发表于 2017-07-16 | 分类于 CSS | 热度
字数统计 2,539 | 阅读时长 9

如何理解元素的浮动以及如何清除浮动的问题，其实已经是老生常谈的问题了，不过，在我这，还是要再讲一遍！为什么？因为，就算是出现频率这么高的问题，在面试的时候依然能被问住！所以，我想好好地总结一番。

浮动及浮动的特征

一句话介绍浮动：浮动元素会脱离文档流并向左/向右浮动，直到碰到父元素或者另一个浮动元素。

下面会通过详细介绍浮动的特征来带大家全面理解浮动！

浮动的特征

1.脱离文档流

这里的文档流，其实在W3C标准中，被叫做普通流（normal flow），但是国内翻译里出现的都是文档流，久而久之就用文档流的说法了。

看上面这个图，以3个div的布局为例。由于div是块状元素，需要独占一行，所以在3个div都不浮动的情况下，应该如左图那样从上至下排列。而右图里，div1左浮动，div2右浮动，所以显示在图里的左上角和右上角，此外他俩脱离了文档流，所以他们后面的元素就自然往上移动，出现了图中绿色div1覆盖了部分蓝色div2面积的情况。

好了，这个大家若懂了，再思考个问题：

浮动元素都脱离文档流了，不是覆盖了下方的元素了吗，“div3”这几个字不应该也是被覆盖了吗，为何这里还是出现了，而且还环绕着div1呢？

这边就要再穿插个小知识点，《精通CSS》这本书中提到：

浮动会让元素脱离文档流，不再影响不浮动的元素。实际上，并不完全如此。如果浮动的元素后面有一个文档流中的元素，那么这个元素的框会表现得像浮动根本不存在一样。但是，框的文本内容会受到浮动元素的影响，会移动以留出空间。

所以呢，其实这个就是float独有的一个作用，能够实现文本环绕图片的效果。

2.浮动可以实现内联排列

浮动会向左/向右浮动，直到碰到另一个浮动元素为止，这是浮动可以内联排列的特征。也就是说，浮动可以设置宽高，并且能够一行多个。

3.浮动会导致父元素高度塌陷

什么是高度塌陷？

当没有指定高度的父元素中的子元素全部都浮动时，父元素中内部高度因为是普通流中的块元素撑起来的，所以这个时候父元素的高度会变成0或者会有部分浮动元素的位置会超出父元素的高度之外，这种现象叫做高度坍塌。给出代码：

浮动会引起的问题

问题主要有几个：

1.父元素高度塌陷，影响与父元素同级的元素。

2.下方非文本元素与浮动元素重合，破坏了原本布局。

针对浮动带来的影响，我们需要清除浮动，以下将介绍清除浮动和几种解决方法：

清除浮动

  • 1.清除浮动并不是让浮动元素回到文档流
  • 2.清除浮动只会改变该元素自身的位置，并且只针对排在该元素前面的元素，目的是让自己的左边或者右边没有浮动元素。

清除浮动的关键字是clear，是css属性，取值如下：

clear : none | left | right | both

  • none : 默认值。允许两边都可以有浮动对象
  • left : 不允许左边有浮动对象
  • right : 不允许右边有浮动对象
  • both : 左右两侧均不允许有浮动对象

使用CSS的:after伪元素

这是目前比较推荐的方法，不会引入无意义的html标签，而是通过:after伪元素在父容器的内部元素最后添加了一个看不见的空格“/20”或点“.” ，并且赋予clear属性来清除浮动，其原理同第一方法。为了兼容IE6和IE7浏览器，要给box-wrapper这个class添加一条zoom:1;触发haslayout（你可以把它写到IE6、7的CSS hacker文件里）。

给父容器添加overflow属性

给包含浮动元素的父元素添加overflow:hidden（或overflow:auto;）

这种方法目前还是挺常用的，但建议在局部和无法添加class的地方使用该方法清除浮动，但是不要作为主要清理浮动方式。

让父容器也浮动

给浮动元素的容器也添加上浮动属性即可清理内部浮动，这种就比较简单，但是，由于父容器和子元素都浮动脱离文档流了，必然会影响紧跟在其后面的元素。

总结

1.一句话介绍浮动：浮动元素会脱离文档流并向左/向右浮动，直到碰到父元素或者另一个浮动元素。

2.浮动的特征：

  • 脱离文档流
  • 实现内联排列
  • 会导致父元素高度塌陷

3.清除浮动（只列出常用的四种，还有更多方法请参考其他文章）

  • 使用带有clear属性的空元素
  • 使用CSS的:after伪元素
  • 给父容器添加overflow属性
  • 让父容器也浮动
