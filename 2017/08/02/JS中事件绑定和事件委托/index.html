事件绑定

事件就是用户或浏览器自身执行的某种动作，如click、load、moseover这些都是事件的名字，而响应某个事件的函数就是事件处理程序。事件处理程序函数的名字以on开头，因此click的事件处理程序就是onclick，诸如此类，还有onload、onmouseover、onmouseout等等。

事件绑定的三种实现方法：

  • 在DOM元素中直接绑定
  • 在JavaScript代码中绑定
  • 绑定事件监听函数

绑定事件监听函数

用 addEventListener() 或attachEvent()来绑定事件监听函数

1.W3C规范

使用addEventListener()函数，接受三个参数 ，其语法：
elementObject.addEventListener(eventName,handler,useCapture);

  • elementObject : DOM对象
  • eventName : 事件名，支持所有DOM 元素。注意：这里的事件名没有“on”，如click、mouseover、doubleclick等
  • handler ：事件句柄函数，即用来处理事件的函数
  • useCapture ： 布尔值，是否使用捕获，一般用false

addEventListener()是标准的绑定事件监听函数的方法，是W3C所支持的，Chrome、FireFox、Opera、Safari、IE9.0及其以上版本都支持该函数；但是，IE8.0及其以下版本不支持该方法，它使用attachEvent()来绑定事件监听函数。

2.IE标准

使用attachEvent()函数，接受两个参数 ，其语法：

elementObject.attachEvent(eventName,handler);

  • elementObject : DOM对象
  • eventName : 事件名，注意：这里的事件名有“on”，如onclick、onmouseover、ondoubleclick等
  • handler : 事件句柄函数，即用来处理事件的函数

事件委托

事件委托把原本需要绑定的额时间委托给父元素，让父元素担当事件监听的职务。可以解决需要把监听事件绑定到大量元素上的情况。利用事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件。

优点：提高JavaScript性能。事件委托可以显著的提高事件的处理速度，减少内存的占用
